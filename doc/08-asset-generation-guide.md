# Asset Generation Guide — LockUp

Version: 1.0  
Status: Creative Production System  
Scope: Icons, illustrations, marketing visuals, UI assets, app store creatives

---

## 1. Purpose

This document defines how all visual assets in LockUp are created.

It ensures:

- Visual consistency
- Premium aesthetic quality
- Emotional alignment with discipline theme
- No random or low-quality visuals
- Unified brand identity across all surfaces

---

## 2. Core Principle

> “Every asset must reinforce discipline or focus.”

If an asset does not:

- Improve clarity
- Reinforce behavior
- Strengthen emotional identity

Then it should not exist.

---

## 3. Asset Style Philosophy

LockUp visuals must feel:

- Calm
- Premium
- Minimal
- Focused
- Slightly emotional but controlled

Inspired by:

- Apple (clarity)
- Linear (precision)
- Duolingo (behavior reinforcement)
- Forest (focus mood)
- Notion (simplicity)

BUT:

Never visually copy any of them.

---

## 4. Icon System

---

### 4.1 Style Rules

Icons must be:

- Line-based or soft filled
- Minimal detail
- Rounded edges
- Consistent stroke width (1.5–2px equivalent)
- No unnecessary decoration

---

### 4.2 Icon Meaning Rules

Icons represent:

- Actions (start, pause, stop session)
- States (active, completed, failed)
- Concepts (focus, discipline, streak, XP)

NOT decorative symbols.

---

### 4.3 Forbidden Icon Styles

- 3D glossy icons
- Overly detailed illustrations
- Cartoon-style icons
- Random gradient-heavy symbols

---

## 5. Illustration System

---

### 5.1 Purpose

Illustrations are used for:

- Emotional reinforcement
- Onboarding explanation
- Empty states
- Achievement moments

---

### 5.2 Style Direction

Illustrations must be:

- Soft gradient-based or flat minimal
- Low complexity
- Emotionally calm
- Slightly abstract

---

### 5.3 Themes

Core themes:

- Focus environment
- Time control
- Discipline growth
- Digital resistance (distraction blocking)
- Progress visualization

---

### 5.4 Example Concepts

- A glowing locked phone surrounded by soft shadows
- A calm student focusing under a floating timer
- A growing progress tree symbolizing streaks
- A shield protecting a focus zone

---

## 6. App UI Assets

---

### 6.1 Buttons & UI Elements

All UI assets must:

- Match design system tokens
- Avoid decorative textures
- Use consistent radius and spacing rules

---

### 6.2 Empty States

Empty states must:

- Encourage action
- Avoid negativity
- Reinforce discipline identity

Example:

Instead of:
> “No sessions yet”

Use:
> “Start your first discipline session”

---

## 7. Marketing Visual System

---

### 7.1 Purpose

Marketing assets must:

- Create emotional urgency
- Communicate discipline identity shift
- Simplify LockUp concept visually

---

### 7.2 Content Types

- App store screenshots
- Social media posts
- Landing page banners
- Feature highlight graphics

---

### 7.3 Design Style

- Dark calm backgrounds
- Strong typography hierarchy
- Minimal UI mockups
- One message per visual
- High contrast focus points

---

## 8. App Store Creative Rules

---

### 8.1 Screenshot Structure

Each screenshot must represent:

1. Problem
2. Solution
3. Lock system
4. XP/streak system
5. Discipline transformation

---

### 8.2 Text Rules

- Short sentences only
- No paragraphs
- Emotional clarity > feature explanation

Example:

> “Block distractions instantly”

> “Build unbreakable focus”

---

## 9. Motion Assets

---

Motion assets are used for:

- Session start animation
- XP gain feedback
- Streak progression
- Completion celebration

Rules:

- Smooth transitions
- No flashy effects
- Motion must feel “earned”

---

## 10. Asset Generation Prompts Standard

All AI-generated assets must follow:

### Prompt Structure:

- Context (what it represents)
- Emotion (focus, discipline, calm)
- Style constraints (minimal, premium, soft)
- Exclusions (no clutter, no cartoon style)

---

### Example Prompt:

> “Minimal illustration of a student focusing at a desk with a glowing timer, calm atmosphere, soft lighting, premium mobile app style, abstract background, no clutter, no cartoon style”

---

## 11. Quality Requirements

Every asset must pass:

- Is it visually calm?
- Does it reinforce discipline?
- Does it match system design tokens?
- Would it fit in a top-tier App Store product?

If not → regenerate.

---

## 12. Anti-Patterns

DO NOT:

- Use stock images
- Use generic emojis as system icons
- Overdecorate illustrations
- Add unnecessary gradients
- Mix multiple visual styles

---

## 13. Final Principle

Assets are not decoration.

They are behavioral reinforcement tools.

In LockUp:

> Every visual must strengthen discipline identity.
